Autocomplete provides the necessary components for you to compose with the other components, **by itself it does not do much**, you need to use the `<ClayDropDown />` along with `<Autocomplete.DropDown />` and `<Autocomplete.Item />` to create the suggestions.

```jsx
<ClayAutocomplete.DropDown>
	<ClayDropDown.ItemList>
		<ClayAutocomplete.Item
			match="cl"
			value="clay"
		/>
	</ClayDropDown.ItemList>
</ClayAutocomplete.DropDown>
```

When you already have the raw data and do not need to consult them, it becomes simpler to create an autocomplete. It is important that you **create a filter of your data** according to the value entered in the input element, following your business rule.

```jsx
<ClayAutocomplete>
	<ClayAutocomplete.Input
		onChange={event => setValue(event.target.value)}
		value={value}
	/>
	<ClayAutocomplete.DropDown active={showDropDown}>
		<ClayDropDown.ItemList>
			{resourceFiltered.map(item => (
				<ClayAutocomplete.Item
					key={item}
					match={value}
					value={item}
				/>
			))}
		</ClayDropDown.ItemList>
	</ClayAutocomplete.DropDown>
</ClayAutocomplete>
```

## Item

`ClayAutocomplete.Item` is a wrapper above `ClayDropDown.Item` but covers the use case to underline the bold characters in case you follow the Lexicon. Feel free to create your own match rule using `ClayDropDown.Item` which will give you more freedom to do so.

<div class="sheet-example">
	<ul class="list-unstyled">
		<li>
			<span class="dropdown-item">
				<div>
					<strong>C</strong>
					<strong>l</strong>
					ay
				</div>
			</span>
		</li>
	</ul>
</div>

```jsx
<ClayAutocomplete.Item
	match="Cl"
	value="Clay"
/>
```

## Loading state

`<ClayAutocomplete.LoadingIndicator />` is used for cases where you need to provide the user with feedback that is loading new data to fill in the suggestion box, it does communicate with `<ClayAutocomplete.Input />` for the markup can adjust.

## Data from outside

For cases where you do not have all the data in the client and need to pass the value for the filter to the backend, use the data provider for this, `<ClayDataProvider />` component or `useResource` hook.
